## Executive Summary

**Verbatim** is a sophisticated web application designed to help users memorize text content using proven cognitive techniques. The app employs a structured three-step memorization methodology: **Familiarize → Encode → Test**. Built with modern web technologies, it provides an intuitive, mobile-friendly interface for creating, managing, and practicing memorization sets.

### Core Value Proposition
- **Structured Learning Path**: Guides users through proven memorization techniques
- **First-Letter Encoding**: Uses cognitive science-backed method for memory retention
- **Flexible Testing**: Multiple test modes to reinforce learning
- **Zero Backend**: Fully client-side with localStorage persistence

### Data Architecture

#### Data Model

```typescript
interface MemorizationSet {
  id: string                    // UUID generated by crypto.randomUUID()
  title: string                 // User-defined title
  content: string              // Original text content
  createdAt: string            // ISO 8601 timestamp
  updatedAt: string            // ISO 8601 timestamp
  chunkMode: "paragraph" | "sentence"  // Chunking strategy
  chunks: Chunk[]              // Parsed chunks
}

interface Chunk {
  id: string                   // UUID for the chunk
  orderIndex: number           // Position in sequence (0-based)
  text: string                // Chunk content
}
```

#### Data Flow

1. **Creation Flow**:
   ```
   User Input → addSet() → generateChunks() → localStorage → UI Update
   ```

2. **Chunk Mode Toggle**:
   ```
   User Selection → updateChunkMode() → Re-parse content → localStorage → UI Update
   ```

3. **Data Persistence**:
   - Stored in `localStorage` under key: `"memorization-sets"`
   - JSON serialized array of MemorizationSet objects
   - Automatic save on any state change
   - Automatic load on app initialization

## Text Processing Utilities

### File: `lib/text-utils.ts`

#### 1. Paragraph Parsing
```typescript
function parseIntoParagraphs(content: string): string[]
```

**Rules:**
- Split on: `\n\s*\n` or single `\n`
- Normalize: Line endings to `\n`
- Trim: Each paragraph
- Collapse: Multiple spaces → single space
- Filter: Remove empty paragraphs

**Example:**
```
Input:  "First paragraph\n\nSecond paragraph\n  Third paragraph"
Output: ["First paragraph", "Second paragraph", "Third paragraph"]
```

#### 2. Sentence Parsing
```typescript
function parseIntoSentences(content: string): string[]
```

**Rules:**
- Split on: `. ` `? ` `! ` (sentence-ending punctuation + space)
- Preserve: Punctuation with sentence
- Normalize: All line breaks → spaces
- Collapse: Multiple spaces → single space
- Filter: Remove empty sentences

**Example:**
```
Input:  "Hello world. How are you? I'm fine!"
Output: ["Hello world.", "How are you?", "I'm fine!"]
```

#### 3. First Letter Extraction
```typescript
function toFirstLetterFormat(text: string): string
```

**Rules:**
- Extract: First letter of each word
- Ignore: Leading punctuation (`"`, `(`, etc.)
- Preserve: Original capitalization
- Skip: Number-only or punctuation-only tokens
- Separator: Single space

**Examples:**
```
"Memory under pressure reveals preparation" → "M u p r p"
"Hello, world! How are you?" → "H w H a y"
"It's a beautiful day." → "I a b d"
```

#### 4. Word Parsing
```typescript
function parseWords(text: string): ParsedWord[]

interface ParsedWord {
  word: string        // Full word with punctuation
  firstLetter: string // Lowercase first letter
}
```

**Purpose:** Convert text to array for encoding practice

#### 5. Text Comparison
```typescript
function compareTexts(typed: string, original: string): ComparisonResult

interface ComparisonResult {
  words: ComparedWord[]       // All words with status
  correctCount: number
  incorrectCount: number
  missingCount: number
  extraCount: number
  totalExpected: number
  accuracy: number           // (correct / totalExpected) * 100
  mistakes: Mistake[]        // Detailed error list
}
```

**Comparison Logic:**
- **Position-based matching**: Word index must match
- **Normalized comparison**: Case and punctuation ignored
- **Four status types**: correct, incorrect, missing, extra
- **Accuracy calculation**: Based only on correct vs. total expected

**Word Classification:**
```
If both words exist at index i:
  If normalized match → CORRECT
  If no match → INCORRECT
  
If only original exists at index i → MISSING
If only typed exists at index i → EXTRA
```

### Core Functions

#### `addSet(title, content)`
**Purpose:** Create new memorization set

**Process:**
1. Generate UUID
2. Create ISO timestamp
3. Default to "paragraph" mode
4. Generate chunks via `generateChunks()`
5. Add to state (prepend to array)
6. Auto-save to localStorage
7. Return new ID

**Usage:**
```typescript
const id = addSet("Hamlet Soliloquy", "To be or not to be...")
router.push(`/memorization/${id}`)
```

---

#### `updateChunkMode(id, mode)`
**Purpose:** Switch between paragraph/sentence chunking

**Process:**
1. Find set by ID
2. Check if mode changed
3. Re-parse content with new mode
4. Update chunks array
5. Update timestamp
6. Auto-save to localStorage

**Effect:** Entire UI updates with new chunks

---

#### `generateChunks(content, mode)`
**Purpose:** Parse content into chunks

**Process:**
1. Call appropriate parser (paragraph or sentence)
2. Filter empty strings
3. Create Chunk objects with:
   - UUID
   - Order index
   - Text content
4. Return array

### localStorage Integration

**Key:** `"memorization-sets"`

**Save Logic:**
```typescript
function saveToStorage(sets: MemorizationSet[]) {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(sets))
}
```

**Load Logic:**
```typescript
function loadFromStorage(): MemorizationSet[] {
  const stored = localStorage.getItem(STORAGE_KEY)
  return stored ? JSON.parse(stored) : []
}
```

**Migration:**
- Old formats without `chunkMode` → Auto-upgraded to paragraph mode
- Missing `updatedAt` → Copied from `createdAt`

**Automatic Sync:**
- `useEffect` watches `sets` state
- Any change triggers `saveToStorage()`
- Only saves after initial load complete (`isLoaded === true`)
